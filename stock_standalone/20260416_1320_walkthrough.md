# 语音播报与信号日志同步修复报告 (20260416_1320)

已修复信号日志在语音播报时不自动滚动定位的问题。本次修改聚焦于恢复“Code 对齐优先”的逻辑，确保 UI 能够实时响应播报进度。

## 主要变更

### [trade_visualizer_qt6.py](file:///d:/MacTools/WorkFile/WorkSpace/pyQuant3/stock_standalone/trade_visualizer_qt6.py)
- **优化反馈轮询**：将用户交互导致的自动滚动锁定阈值从 `3.0s` 降低至 `1.5s`。
- **提升灵敏度**：确保在播报开始时，系统能更快速地判断是否需要执行滚动。

### [signal_log_panel.py](file:///d:/MacTools/WorkFile/WorkSpace/pyQuant3/stock_standalone/signal_log_panel.py)
- **强化滚动定位**：在 `highlight_row_by_content` 中引入了 `PositionAtTop` 滚动提示，确保选中的信号条目始终显示在表格顶端，视觉效果更直接。
- **恢复 Code 锚点逻辑**：保留了对 Code 的精准定位，确保即使日志内容发生合并或去重，也能准确定位到对应的个股行。

## 验证结果

- [x] **语音播报联动**：播报开始时，日志面板能自动选中对应股票并滚动至显眼位置。

- [x] **多行匹配测试**：对于同一股票多次触发的信号，播报能正确指向最新的日志行。
