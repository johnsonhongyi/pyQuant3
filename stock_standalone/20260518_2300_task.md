# 20260518_2300_task - 修复次新股极短 K 线切换到正常个股的 viewport 错位 bug

## 1. 任务背景与核心痛点
- **背景**：当用户在 `trade_visualizer_qt6.py` 中切换股票时，如果前一个股票是次新股/上市初期股票（K 线数据量极少，例如少于 35 根 K 线），再切换到数据量正常的正常个股时，K 线视口（Viewport）会发生严重错位：没有在右侧最新处对齐，而是停留在了最左侧/最旧的 K 线，用户需要手动拖拽或点击 Reset 才能看到最新行情。
- **痛点原因分析**：
  1. 在旧的 `_capture_view_state` 方法中，当 `len(self.day_df) < 30` 时，函数会直接提前截断并 `return`。
  2. 这导致当切换走时，完全没有设置诸如 `_prev_total_bars` 或任何标志来识别“前序个股属于极短次新股”。
  3. 当加载新正常股时，`_prev_dist_left` 和 `_prev_dist_right` 等变量未被清理，导致系统错误地将其判定为“有可用记忆视口”，从而应用了非标的极短视口比例（或者完全越界），使得视口拉大到了最左侧的极旧区域。
  
## 2. 解决方案设计与实施
我们严格遵循 KISS 与 SOLID 原则，实施了以下手术级的精密修复：

1. **重构 `_capture_view_state`**：
   - 提取当前股票的 `total` 数据长度，记录 `self._prev_total_bars` 与 `self._prev_is_very_short` 状态。
   - 引入专用标志 `self._prev_kline_too_short = (total < 35)`，当离开极短次新股时，主动删除全部缓存的视口属性（`_prev_dist_left` 等），确保不会被当成“有效的视口记忆”。
   - 随后提前 `return`，完全防止了越界与干扰。
   
2. **升级 `_render_charts_logic` 中的视口重置与判定机制**：
   - 精准捕获 `prev_too_short = getattr(self, '_prev_kline_too_short', False)` 标志。
   - 当 `prev_too_short` 为 `True` 且新加载的个股为正常长度时（`len(day_df) >= 35`），立刻清除该标记，并强制调用 `_reset_kline_view(df=day_df, force=False)`，彻底将 X 轴视口以最高优先级右对齐最右侧最新行情，消除了任何错位。
   - 在常规 `elif is_new_stock...` 段中，将 `prev_too_short` 和 `is_prev_short` 全面绑定，进一步加固多重阻断防线。
