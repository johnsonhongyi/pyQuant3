## 交易时段定义

```python
# 同时检查交易日和交易时段
is_trade_day = cct.get_trade_date_status()  # 是否为交易日
now_time_int = cct.get_now_time_int()  # HHMM 格式整数
is_trading_time = (930 <= now_time_int <= 1130) or (1300 <= now_time_int <= 1500)

# 必须同时满足两个条件
if not is_trade_day or not is_trading_time:
    return  # 跳过写入
```

**交易时段**:
- 上午: 09:30 - 11:30
- 下午: 13:00 - 15:00

**非交易时段**(将被过滤):
- **非交易日**: 周末、节假日等
- 集合竞价: 09:15 - 09:30
- 午间休市: 11:30 - 13:00
- 收盘后: 15:00 之后
- 开盘前: 09:30 之前

## 技术细节

### 时间格式标准

**统一格式**: `YYYY-MM-DD HH:MM:SS`

**示例**: `2026-02-11 14:30:25`

**优点**:
- 便于排序和查询
- 包含完整日期信息
- 与其他信号格式一致
- 支持跨日分析

### 异常处理策略

**保守策略**: 当交易时段检查失败时,允许写入信号

**理由**:
- 避免因检查逻辑问题导致信号丢失
- 确保系统稳定性
- 可通过定期清理工具补救

### 性能影响

**交易时段检查**:
- 时间复杂度: O(1)
- 性能影响: 可忽略不计
- 每次写入增加约 0.1ms

**清理工具**:
- 批量删除,效率高
- 大量数据(10万条)约需 5-10 秒
