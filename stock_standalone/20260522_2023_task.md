## 1. 任务背景 (Background)
在“策略选股与实时决策”主终端中，下半部分是专为交易员设计的数据复盘和监控分栏，由两个主要 Tab 选项卡构成：
1. **📊 当前持仓 Tab** (`self._pos_tree`)：呈现交易员当前的实时股票持仓、盈亏及止损数据；
2. **📜 今日流水 Tab** (`self._log_tree`)：呈现今日买入、卖出的流水记录与触发原因。

**存在痛点**：
- 以前这两个表格虽然能展示数据，但是**缺乏任何事件交互与代码联动**。
- 交易员如果想看一下当前某个持仓股的走势，或者想审查流水中某只股票的买卖点，必须手动记住代码并在其它窗口中输入搜索。这在交易瞬息万变的临盘和高频复盘场景下，存在极高的人机摩擦与时间损耗。

**优化目标**：
- 让这两个 Tab 下的表格全部支持**一键极速联动刷新 K 线主图及可视化面板**！
- 达成“点到哪儿、联到哪儿”的高级极客操作体验，免除任何人工手动查找的繁琐。

## 2. 联动数据流与事件捕获设计 (Data & Event Design)
1. **高精代码捕获 (O(1) Complexity)**：
   - **当前持仓表 (`self._pos_tree`)**：在数据装载写入时，每一行的唯一标识符 `iid` 被直接设定为了股票代码本身。我们只需通过 `sel = self._pos_tree.selection()` 并提取 `sel[0]`，即可在 $O(1)$ 复杂度内获得补全的 6 位股票代码。
   - **今日流水表 (`self._log_tree`)**：由于可能存在同股多次流水的关系，其 `iid` 为递增序号。但是其 `values` 的第三列（即 `values[2]`）存储了对应的个股代码。我们通过 `item = self._log_tree.item(sel[0])` 并读取其 values 数组索引为 2 的值即可瞬间提取。
2. **双通道同步发送机制**：
   - **主 K 线联动**：提取到个股代码后，调用 `self.sender.send(code)` 瞬间在后台通过通道向主图发送，实现主图自动同步切股重绘；
   - **可视化子进程自愈联动**：同步判定主控制台的 `vis_var` 变量（即“联动可视化”是否勾选）。如果处于勾选激活状态，级联调用 `self.master.open_visualizer(code)` 投递，使联动可视化进程在亚毫秒级内自动更新视角。
3. **极速交互绑定 (Single-click Interaction)**：
   - 为持仓和流水表格的 `<<TreeviewSelect>>` 事件分别动态绑定我们新增的联动驱动接口，实现用户一旦使用鼠标或键盘上下键切换选择行时，全系统以极高敏捷度同步刷新。

```python
# stock_selection_window.py (File End)

def _on_pos_selected(self, event=None):
    """当前持仓行选中联动主图与 K 线可视化"""
    try:
        sel = self._pos_tree.selection()
        if not sel:
            return
        code = str(sel[0]).zfill(6)  # iid 就是个股代码本身
        if hasattr(self, 'sender') and self.sender:
            try:
                self.sender.send(code)
            except Exception:
                pass
        if getattr(self, 'master', None) and getattr(self.master, "vis_var", None) and self.master.vis_var.get():
            if hasattr(self.master, 'open_visualizer'):
                self.master.open_visualizer(code)
    except Exception as e:
        logger.debug(f"[on_pos_selected] error: {e}")


def _on_log_selected(self, event=None):
    """今日流水行选中联动主图与 K 线可视化"""
    try:
        sel = self._log_tree.selection()
        if not sel:
            return
        item = self._log_tree.item(sel[0])
        vals = item.get('values', [])
        if len(vals) > 2:
            code = str(vals[2]).zfill(6)  # values[2] 是个股代码
            if hasattr(self, 'sender') and self.sender:
                try:
                    self.sender.send(code)
                except Exception:
                    pass
            if getattr(self, 'master', None) and getattr(self.master, "vis_var", None) and self.master.vis_var.get():
                if hasattr(self.master, 'open_visualizer'):
                    self.master.open_visualizer(code)
    except Exception as e:
        logger.debug(f"[on_log_selected] error: {e}")


StockSelectionWindow._on_pos_selected       = _on_pos_selected
StockSelectionWindow._on_log_selected       = _on_log_selected
```

## 4. 核心设计原则遵循 (SOLID)
- **单一职责原则 (SRP)**：解耦了数据展示与数据通信。表格只负责渲染和维持状态，具体的联动行为移交给独立的事件回调，通信通过轻量的 `sender` 通道进行投递，非常纯粹。
- **杜绝重复 (DRY)**：针对持仓代码与流水代码的特殊结构分别编写最简的数据解包方案，并完美复用了系统既有的 K 线主图及 `open_visualizer` 多进程可视化联动管道，以极低的代码量实现全终端高精交互。
