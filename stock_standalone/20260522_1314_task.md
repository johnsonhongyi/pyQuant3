## 落地多级实时行情自愈补齐机制，彻底攻克增量冷启动跟随股“无分时图及0.00元价格”问题 (Implemented Multi-level Real-time Data Healing for Lagging Followers)

### 1. 痛点与深度机理解析
- **业务表现**：在昨日收盘次日冷启动、或系统刚运行的活跃期，虽然龙头股和高分股的分时走势正常，但是大批现价显示为 `0.00` 的普通跟随股（如 衢州东峰、捷佳伟创、经纬辉开、璞泰来、远东股份 等）却始终空无分时走势图。
- **罪魁祸首——增量打分与持久池的“冷启动空洞”**：
  - 为了杜绝全量计算带来的严重卡顿，系统引入了高效的增量打分机制：非 essential（核心活跃）的弱势股不参与每轮的高频行情拉取和打分更新。
  - 这些跟随股在开盘后由于得分跌破门槛，不再处于 essential 更新名单中，因而其数据一直停留在开盘初始化时的“初始状态”中（现价为 `0.00`，分时 `prices` 为空列表，昨收 `last_close` 也是 `0.00`）。
  - 因为数据一直没有被增量分组更新，所以它们在持久化池 `self._sector_active_stocks_persistent` 中成为了“僵尸行”，永远无法生成今日分时走势，使得表格里出现大面积惨白。

### 2. 极致的自愈解决方案
- **数据源头循环入口数据覆写补齐**：
  - 在 `sector_bidding_panel.py` 的个股表格渲染迭代入口处（`for i, r in enumerate(rows)`），新增了高精度的多级实时内存行情拦截与自愈补齐机制：
    ```python
    if (r.get('price', 0) <= 0 or not r.get('klines') or r.get('last_close', 0) <= 0) and hasattr(self, 'detector') and self.detector:
        # 第一级自愈：从 detector 的全局行情快照缓存 (global_snap_cache) 补齐
        ...
        # 第二级自愈：从 TickSeries 实体存储中拉取补齐
        ...
    ```
  - **自愈内容**：
    - **现价 (Price)**：将 `0.00` 覆盖为最新的真实非零现价。
    - **昨收 (Last Close)**：若为 0，自动补齐为高频行情商返回的昨收价。
    - **分时走势 (K-lines)**：若 `klines` 为空，直接从全局行情中解包并补齐分时线、均线、成交量走势。
    - **涨幅 (Pct)**：如果缺省，利用补齐后的现价和昨收自动重新校准涨幅！
