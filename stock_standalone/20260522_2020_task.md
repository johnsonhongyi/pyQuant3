## 1. 任务背景 (Background)
用户在使用“策略选股与确认”主终端以及下方的“🎯 实时买点决策队列”时，发现在没有数据或者数据行较少时，整个表格下方的大片多余空白区域会显示为刺眼的纯白色。
此外，Windows 系统的 Treeview 表格控件默认带有高亮的亮灰色/白色立体边框（即白框）。
这两项系统默认行为：
1. **视觉硬伤**：大面积的惨白底色和白框直接破坏了系统精心打造的高反差、夜间暗黑系极客外观风格；
2. **数据显示割裂**：有数据的时候，普通数据行之前被写死为了大白色背景；没数据的时候也是大白色，导致数据凸显性与融合感全无。

为了达成极客级的交互终端质感，用户提出硬性标准：
- **有数据跟数据背景同色**：有数据的行、无数据时的空白背景，必须全部统合、完全一模一样地渲染为统一的深黑暗底色 `#0c101b` / `#0a0f1a`！
- **不显示白边框**：物理移除和隐藏所有 Treeview 控件的默认立体白色边框。

## 2. 扁平化暗黑色彩与防变形安全兜底设计 (Styling & Fail-Safe Layout Design)
1. **彻底拒绝全局主题污染 (Anti-Deformation Guard)**：
   - 之前为快速渲染背景曾引入了全局 `style.theme_use("clam")` 配置。
   - **痛点**：因为 `theme_use` 在 `ttk` 中是全局生效的，一旦调用，主程序窗口以及其他界面的所有默认 `ttk` 控件（按钮、输入框、下拉框、状态栏）全都会被强制变更为 `clam` 主题样式，从而引发严重的全局排版错位与布局变形！
   - **解决设计**：我们**彻底物理剔除了所有的全局主题切换语句** (`theme_use("clam")`)。这百分之百确保了整个 Tk 主程序和所有子对话框绝对不会由于选股窗口的初始化而发生任何意料之外的布局变形，保持了系统一贯的原生高保真和稳定性。
2. **深度定制 `Custom.Treeview` 样式 (平滑降级安全兜底)**：
   - 配置 `background="#0c101b"`：有数据时，数据所在行的默认底色为深黑；
   - 配置 `fieldbackground="#0c101b"`：无数据时，整个数据填充和余量填充区域自动渲染为完全一致的深黑底；
   - 配置 `foreground="#ffffff"`：数据前景色为亮白色，文字识别度直线上升；
   - 配置 `borderwidth=0, highlightthickness=0`：将物理边框和外侧焦点虚线彻底降为 0，彻底根除白色大边框。
   - **平滑自愈**：在 vista/xpnative 等默认 Windows 主题下，自定义样式会自动作为平滑降级（Graceful Degradation）的安全兜底配置，绝对不会产生任何运行期异常或错位。
3. **清理残留白底行配置 (Tags Alignment)**：
   - 将 Tab 1 中写死为大白背景的普通待复核行标签 `tag_configure("pending", background="#ffffff")` 修正为与背景同色的 `#0c101b`；
   - 将概念详情表格中高命中标签 `high_hits` 背景由 `#e8f5e9` 修正为暗绿色的微绿底色 `#13261a`，消除了行高亮时的亮色刺眼感。

## 3. 代码变更详情 (Source Diffs)

### 3.1 统一定制 Custom.Treeview 扁平化全局样式 (无全局 theme_use)
```python
# stock_selection_window.py -> _init_ui
        # 深度定制 Custom.Treeview 样式 (有数据背景与数据填充区域完全同色)
        style.configure("Custom.Treeview",
                        background="#0c101b",
                        foreground="#ffffff",
                        fieldbackground="#0c101b",
                        rowheight=22,
                        borderwidth=0,
                        highlightthickness=0,
                        font=("Arial", 9))
        
        # 定制表头样式
        style.configure("Custom.Treeview.Heading",
                        background="#111726",
                        foreground="#ffffff",
                        relief="flat",
                        borderwidth=0,
                        highlightthickness=0,
                        font=("Arial", 9, "bold"))
        
        # 映射选中时的背景与前景
        style.map("Custom.Treeview",
                  background=[("selected", "#1a302a")],
                  foreground=[("selected", "#55ffaa")])
```

## 4. 核心工程原则遵循 (SOLID / KISS)
- **KISS原则 (KISS)**：拒绝引入不必要的复杂度。在接到用户“防变形”反馈后瞬间物理剔除全局 `theme_use` 主题切换，在保持系统最高级原生排版保真度的前提下，以最小的变更幅度和最高的安全系数完成了优雅的背景渲染控制。
