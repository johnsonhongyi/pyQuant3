## 修复竞价赛马面板首屏与行情更新中个股总数不对齐及无效占位数据问题 (Fixed Stock Count Discrepancy & Sanitized Invalid Placeholders in df_all)
**Date:** 2026-05-06 09:55

### 目标 (Objectives)
修复在主监控程序（TK主视图）中有效 A 股/个股数量（如5493只）与竞价赛马面板（Racing Panel）加载的个股数量（5583只）不对齐的问题。查明原因为 `df_all` 作为全局行情数据源未在数据源头做数据清洗，包含大量已退市、代码为 `000000` 或 `000003`、`000005` 等空名字的无效占位数据。要求在 `instock_MonitorTK.py` 数据入场源头对 `df_all` 进行轻量、彻底的过滤与净化。

### 实施方案 (Implementation Plan)
- [x] **源头级个股数据净化**：在 `instock_MonitorTK.py` 的异步行情接收与净化函数 `_sanitize` 内部，在对数据进行代码格式校对后，加入对股票名称、代码有效性的精密拦截过滤逻辑。
    - [x] 过滤掉所有 `name` 为空、NaN、或者为 `nan`/`NaN`/`None`/`null`/`δ֪`/`未知` 的占位无效数据行，并自动去除名字两端的空白。
    - [x] 过滤掉所有 `code` 长度不为 6、包含非数字，或者为 `000000` 等虚无代码的异常行。
- [x] **数据同步对齐**：此举直接在数据最底层源头完成了净化，所有下游面板（包括竞价赛马板块、领军个股列表、以及重点观察池）在同步 `self.df_all` 或通过 IPC 联动读取最新快照时，均实现 100% 正规有效 A 股的严密对齐。
- [x] **更新开发文档**：同步创建本任务清单并更新 `gemini.md` 中的【变更日志】。
