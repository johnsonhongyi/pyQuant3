# 20260519_0020_task - 在 K 线突破收盘价处绘制向右延伸的价格支撑压力水平线及最右侧数值标记

## 1. 任务背景与核心痛点
- **背景**：用户在看盘时，当股票触发 `pbreak == 1`（平台突破）时，希望能够直观看到所有历史突破收盘价的横向水平延长线（延伸到视口最右侧），并有清晰的价格数值标签展示。
- **痛点**：
  1. 此前虽然绘制了平台顶底通道（`ptop` / `pbottom`），但它们是动态波动的折线，缺乏固定历史突破点收盘价的延长阻力支撑参考。
  2. 画面最右侧没有显示具体的 `ptop`（阻力位价格）和 `pbottom`（支撑位价格）数值标签，用户无法一眼得出精确的具体风控挂单数值。
  3. **首版反馈优化点**：第一版标记的位置处于 `total + 1.1`，由于超过了视口范围边界被右轴遮挡。同时，水平延长线的颜色对比度不够突出，容易被复杂的蜡烛图与背景色掩盖。
  4. **二次反馈需求点**：需要为这一组支撑压力水平延长线和最右侧标记设计一个界面级别的显示状态开关（Checkbox），并与现有的 UI 配置（`visualizer_layout.json`）对接，实现跨会话的开关状态自动持久化。
  
## 2. 解决方案设计与实施
我们严格遵循 KISS 与 SOLID 原则，对渲染管道与 UI 控制层进行了深度的打磨与升级：

1. **新增 `_draw_breakout_price_lines` 绘制管道与极致对比度优化**：
   - 在 `trade_visualizer_qt6.py` 中实现了 `_draw_breakout_price_lines(self, x_axis, day_df, ptop_vals, pbottom_vals)` 函数。
   - **突破点水平价格线**：扫描最近 150 天的历史，针对每一次 `pbreak == 1` 且 `pdays == 1`（突破发生的黄金第一天），以该日收盘价为 $y$ 坐标，绘制一条从突破日起点 `x = x_axis[i]` 延伸至最后一根 K 线位置 `x = total - 1` 的高亮金色虚线。
   - **高亮对比度**：延长线颜色升级为高饱和的鲜黄色（`rgba(255, 215, 0, 240)`），画笔粗度由 `1.0` 增加至 `1.5`，对比度极佳，极易识别。
   - **突破点价格最右侧标记**：在最后一根 K 线 `x = total - 1` 处以 `TextItem` 标记具体的突破价格（例如 `🎯 23.45`），金黄字加上 `rgba(20, 20, 20, 230)` 高不透明度暗色底框与 `1.5px` 黄金描边，完美贴合在延长线右端，既不会被右轴裁剪，又起到了极致的视觉对齐。
   - **阻力支撑实时价格标记**：在 `x = total - 1` 处，分别在最新 `ptop` 阻力位（洋红色 `#FF00FF`，阻力: 23.95）与 `pbottom` 支撑位（亮青色 `#00FFFF`，支撑: 21.30）位置绘制高对比度数值框，实现了“一图知风控”，杜绝了裁剪与遮挡。

2. **添加“支撑压力线”工具栏开关与即时刷盘逻辑**：
   - **植入 Checkbox 开关**：在主工具栏的“突破天数”复选框正后方，平滑植入了 `QCheckBox("支撑压力线")`（变量为 `self.cb_show_breakout_lines`），支持即点即显隐。
   - **槽函数快速重绘与主动保存**：编写了 `_on_toggle_breakout_lines(self, state)` 槽函数，在勾选状态变动时瞬间对 `self.show_breakout_lines` 状态位赋值，并主动调用 `self._save_visualizer_config()` 实现秒级写盘，随后调用 `force=True` 重绘图表，实现所见即所得。
   - **绘图端短路拦截**：在 `_draw_breakout_price_lines` 顶部的初始化与隐藏重置块之后，引入了 `if not getattr(self, 'show_breakout_lines', True): return` 判断，确保关闭时立刻隐藏所有线段与气泡并安全返回。

3. **双向持久化配置绑定 (`_load_visualizer_config` & `_save_visualizer_config`)**：
   - **读取端绑定**：在 `_load_visualizer_config` 中，增加了对 `show_breakout_lines` 属性的安全读取和解包逻辑，并在构造后通过 `blockSignals` 静默套用至复选框 `self.cb_show_breakout_lines`。
   - **写入端绑定**：在 `_save_visualizer_config` 的 window 节点中完美补齐了对 `show_breakout_lines` 的输出，跨会话记忆率达 100%。

4. **清空与防泄露处理 (`_clear_platform_breakout`)**：
   - 彻底升级了清理机制。切换股票时，除隐藏主通道外，一并遍历隐藏所有生成的 `pbreak_price_lines_pool`、`pbreak_price_labels_pool` 以及 `ptop/pbottom` 价格标签，确保零 CPU 泄露与完美冷启动。
