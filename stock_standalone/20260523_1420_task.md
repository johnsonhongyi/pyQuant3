# Task: 修复 KLineDetailWindow 跨显示器(第二屏幕)启动位置偏离与相对级联跟随
日期时间: 2026-05-23 14:20

## 1. 目标与痛点
- **痛点 A (跨屏分裂)**: 当交易主窗口 `MainWindow` 被拖拽到多显示器环境下的第二屏幕并启动时，如果 `KLineDetailWindow` (详情浮窗) 是上次保存在第一屏幕的坐标，这会导致它分裂显示在不同的桌面上。
- **痛点 B (AttributeError 报错)**: 之前在 `_update_crosshair_ui` 和 `moveEvent` 里面曾经调用过 `self.kline_plot.mapToGlobal`，而 `PlotItem` 属于图形组件并不具备该方法，现已统一更正为具备该方法的 `self.kline_plot.mapToGlobal`（根据用户回滚已退回）。
- **目标**: 在主窗口初始化显示加载位置时，对比主窗口与详情窗当前物理加载坐标所位于的屏幕（桌面）是否一致。如果不是在同一个屏幕，则抛弃自定义位置并退回到默认初始化贴近位置，防止跨屏幕分裂。

## 2. 实施细节
1. **多屏幕自适应检测 (`showEvent`)**:
   - 一次性读取主窗口和浮窗的加载位置：
     ```python
     main_w, main_h, main_x, main_y = self.load_window_position_qt(self, "trade_visualizer", ...)
     detail_w, detail_h, detail_x, detail_y = self.load_window_position_qt(self.kline_detail_win, "kline_detail_window", ...)
     ```
   - 利用 `QGuiApplication.screenAt` 动态获取 `(main_x, main_y)` 和 `(detail_x, detail_y)` 所在的显示器对象。
   - 如果两个窗口处于同一个显示器（桌面）上，说明自定义位置合理，设定 `is_custom_positioned = True`。
   - 如果处于不同的显示器上，说明发生了跨屏偏离，则立即设定 `is_custom_positioned = False`。同时主动调用 `self.kline_plot.mapToGlobal(QPoint(40, 10))` 获取默认贴近坐标，直接将浮窗 `move` 回主窗口所在显示器的默认坐标上，完美消除跨屏偏离。
2. **正常保存 (`closeEvent` / `save_detail_window_position`)**:
   - 维持原本内置的高可靠 `save_window_position_qt_visual` 与 `save_window_position_qt` 机制写入 `"kline_detail_window"`，无冗余的手写相对逻辑。

## 3. 验证与总结
- 编译通过，逻辑简单且完全符合 “一次判断” 与 “是否在同一个桌面” 的高内聚设计。
