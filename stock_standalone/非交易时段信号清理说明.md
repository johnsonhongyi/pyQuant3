# 非交易时段信号清理工具使用说明

## 概述

本工具用于清理数据库中所有非交易时段(9:30-11:30, 13:00-15:00)的信号记录,并确保程序后续只在交易时段内写入信号。

## 修改内容

### 1. 新增清理工具

**文件**: `cleanup_non_trading_signals.py`

这是一个独立的命令行工具,用于清理历史数据中的非交易时段信号。

**功能特性**:
- 支持试运行模式(`--dry-run`),只统计不删除
- 自动备份数据库(可通过 `--no-backup` 禁用)
- 支持指定单个数据库或批量处理
- 清理两个主要表:
  - `live_signal_history` (在 `trading_signals.db` 中)
  - `signal_message` (在 `signal_strategy.db` 中)

**使用示例**:

```bash
# 1. 试运行模式(推荐首次使用)
python cleanup_non_trading_signals.py --dry-run

# 2. 清理所有数据库(默认会先备份)
python cleanup_non_trading_signals.py --all

# 3. 只清理 trading_signals.db
python cleanup_non_trading_signals.py --db trading_signals.db

# 4. 清理时不备份(谨慎使用)
python cleanup_non_trading_signals.py --all --no-backup
```

### 2. 程序修改

## 交易时段定义

**检查条件**:
1. **交易日判断**: `is_trade_day = cct.get_trade_date_status()`
2. **交易时段判断**: `is_trading_time = (930 <= now_time_int <= 1130) or (1300 <= now_time_int <= 1500)`

**必须同时满足**: 交易日 AND 交易时段

**交易时段**:
- 上午: 09:30:00 - 11:30:00 (930 - 1130)
- 下午: 13:00:00 - 15:00:00 (1300 - 1500)

**非交易时段**(将被过滤):
- **非交易日**: 周末、节假日等
- 集合竞价: 09:15 - 09:30
- 午间休市: 11:30 - 13:00
- 收盘后: 15:00 之后
- 开盘前: 09:30 之前

## 注意事项

### 1. 数据备份
- 首次使用前务必手动备份数据库
- 工具默认会自动备份,但建议保留多个版本

### 2. 交易日判断
- 程序会同时检查是否为交易日和交易时段
- 非交易日即使在时间范围内也不会写入

### 3. 异常处理
- 如果交易时段检查失败,程序会采用保守策略(允许写入)
- 这样可以避免因检查逻辑问题导致信号丢失

### 4. 性能影响
- 交易时段检查非常轻量,对性能影响可忽略
- 清理工具处理大量数据时可能需要几秒钟

## 故障排查

### 问题1: 清理后仍有非交易时段信号

**可能原因**:
- 清理后程序又写入了新的非交易时段信号
- 数据库文件路径不正确

**解决方法**:
1. 确认程序已重启,加载了新的代码
2. 检查数据库文件路径是否正确
3. 查看程序日志,确认交易时段检查是否生效

### 问题2: 交易时段也没有信号写入

**可能原因**:
- 交易时段判断逻辑错误
- 系统时间不正确

**解决方法**:
1. 检查系统时间是否正确
2. 查看日志中的时间判断信息
3. 临时注释掉交易时段检查,确认其他逻辑正常

### 问题3: 数据库备份失败

**可能原因**:
- 磁盘空间不足
- 数据库文件被占用

**解决方法**:
1. 检查磁盘空间
2. 关闭所有访问数据库的程序
3. 使用 `--no-backup` 跳过自动备份(需手动备份)
