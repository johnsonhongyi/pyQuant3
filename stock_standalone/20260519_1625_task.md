# 20260519_1625_task - Nuitka 打包配置文件 (Console 模式) 生成与优化

## 1. 任务背景与核心需求
- **背景**：项目原本使用 PyInstaller 进行打包，配置定义在 `instock_MonitorTK.spec` 中。用户要求按照该 `.spec` 的配置，生成 Nuitka 的打包配置文件，并采用 **Console 模式** 进行打包。
- **核心需求**：
  1. 完美迁移 `.spec` 中的所有静态数据文件（`datas`），特别是动态解析 `a_trade_calendar` 包路径下的 `a_trade_calendar.csv`。
  2. 补齐所有的隐式导入（`hiddenimports`），确保 Nuitka 编译后的二进制包在运行时不会因为动态导入而报错。
  3. 实现对 `trash_list`（冗余 Qt6 模块与 DLL，如 WebEngine、Qml、Quick、Bluetooth 等）的物理过滤与排除，以极度压缩打包文件体积。
  4. 支持 Nuitka 的 **Tkinter 插件** 和 **PyQt6 插件**，确保主窗口与 K线可视化子进程/对话框均能正常加载。
  5. 设定为 **Console 模式** (`--windows-console-mode=force`)，便于开发与调试时抓取标准输出和错误日志。

## 2. 解决方案设计与实施
为保证最佳的工程实践和多途径打包体验，我们提供两种对齐配置：

### 方案 A：独立编译批处理脚本 `nuitka_build_console.bat`（推荐）
我们新建了一个专门的批处理脚本 `nuitka_build_console.bat`，包含：
1. **自动清理干扰环境**：在 PATH 中排除 Scoop、Git、Anaconda 的 GCC/MinGW 干扰项，确保使用的是 `D:\mingw64` 下的编译器。
2. **动态解析 CSV 路径**：在脚本内自动执行 `%PYTHON_EXEC%` 代码动态提取 `a_trade_calendar.csv` 的绝对路径，杜绝硬编码问题。
3. **数据文件（Data Files）映射**：将所有在 `.spec` 中定义的资源逐一映射为 `--include-data-files` 选项。
4. **隐式导入（Hidden Imports）对齐**：使用 `--include-module` 参数强制包含 NumPy, Pandas, pandas_ta, talib 等。
5. **体继续限压缩（Exclusions）**：
   - 使用 `--exclude-module=PyQt6.QtXXX` 避免导入无效的 Python 绑定。
   - 使用 `--noinclude-dlls=Qt6XXX.dll` 在物理层阻断多余的 C++ DLL 拷贝。
6. **Console 模式配置**：指定 `--windows-console-mode=force` 强制启用控制台。

### 方案 B：嵌入式项目选项 `# nuitka-project:`
我们将所有的配置以符合 Nuitka 标准规范的 `# nuitka-project:` 指令写入到主入口文件 `instock_MonitorTK.py` 的顶部。这样，用户只需直接运行以下命令即可编译：
```bash
python -m nuitka instock_MonitorTK.py
```
