# 任务清单 (2026-05-22 15:52) - 修复 VolumeDetailsDialog 表格白框与多余白列问题

## 1. 问题分析 (Problem Analysis)
- **弹窗整体惨白**：由于 `VolumeDetailsDialog` (继承自 `QDialog`) 自身在初始化时没有设置自定义的深色背景样式，在 Windows 的默认浅色主题环境下，它的背景色会默认为浅灰/白色。这导致顶部的 `header_frame` 说明栏区域以及 "DNA审计" 按钮的周围呈现出刺眼的白色底色，破坏了全终端 sleet/dark（暗黑炫酷）的整体视觉一致性。
- **表头空白区白底**：当窗口横向拉大时，由于表格只有 4 列且没有指定拉伸，表格的右半部分成为空余区。而 `QTableWidget` 默认的 `QHeaderView`（表头）的背景颜色在空白区域为白色，这造成了表格表头右侧出现一个大白块（白框）。
- **右侧多余空列**：由于没有设置 `h_header.setStretchLastSection(True)`，窗口拉大后列宽保持原样，导致列与窗口右侧边界之间出现视觉空隙，显得不够精致。

## 2. 解决方案与实施计划 (Implementation Plan)
1. **加固窗口自身深色背景**：
   在 `VolumeDetailsDialog.__init__` 中为 `QDialog` 指定极客深蓝背景色：
   `self.setStyleSheet("QDialog { background-color: #1a1e2b; color: #ffffff; }")`
   这能让顶部说明栏和按钮的背景瞬间化为协调的深色。
2. **重塑表头空白区域样式**：
   在表格的 QSS 样式表 `self.table.setStyleSheet` 中显式重写 `QHeaderView` 本身：
   ```css
   QHeaderView {
       background-color: #1a1c2c;
       border: none;
   }
   ```
   这彻底终结了表头未用区域显示为默认白色的现象。
3. **启用最后一列自动拉伸**：
   为水平表头 `h_header` 加上 `h_header.setStretchLastSection(True)` 属性，在横向变宽时，让最后一列（“量比”）自适应延展以覆盖多余的空间。

## 4. 设计原则的体现 (Design Principles Application)
- **KISS (保持简单)**：直接利用 Qt 本身提供的 QSS 和 `setStretchLastSection` 机制解决布局和背景问题，无需编写多余的 `resize` 事件计算代码或新增额外的层级布局，代码十分清晰。
- **SOLID (单一职责/开放封闭)**：对 `VolumeDetailsDialog` 的美化和拉伸设置完全封装在其自身的构造函数中，不影响外部行情推送、持久化列宽或排序逻辑，保持了高内聚性。
- **Aesthetic Excellence (高级美学)**：深度贯彻极客暗黑风，通过精准的 `#1a1e2b` 与 `#1a1c2c` 色调搭配，配合最后一列无感延展，将 Windows 系统默认白框/白底完全净化，显著提升界面档次。
