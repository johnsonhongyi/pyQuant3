# 2026-05-30 15:10 Nuitka打包批处理脚本同步支持3个选项任务

## 任务背景与目标
用户希望在 `nuitka_build_console.bat` 和 `nuitka_build_console_onlyClang.bat` 两个批处理脚本中，同步将打包模式选择器从原有的 2 个选项扩展为 3 个选项，使得打包时能支持：
1. **Standalone Folder 独立文件夹模式**：使用 `--standalone`。
2. **Onefile Executable with fixed Temp Dir 带固定临时解压路径的单文件模式**（旧默认）：使用 `--onefile --onefile-tempdir-spec="{TEMP}\instock_Nuitka"`。
3. **Onefile Executable (Standard) 标准单文件模式**（不带解压规约）：仅使用 `--onefile`。

这样不仅解决了 Nuitka 打包路径可能存在的不同需求，也让用户具备更多的选择。

## 工程与设计原则 (KISS, YAGNI, DRY, SOLID)
1. **KISS (简单至上)**：在 Batch 脚本中以最直观易懂的选择逻辑（`choice /C 123`）处理 3 个选项，不做无谓的复杂度扩充。
2. **YAGNI (精益求精)**：只实现用户要求的 3 个选项流，并做对应的命令行参数适配，不加额外的无关功能。
3. **DRY (杜绝重复)**：两份批处理文件的选择器逻辑保持同步对齐，行为高度一致，变量和验证逻辑完美衔接。
