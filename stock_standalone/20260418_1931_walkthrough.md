# Walkthrough: DNA 专项审计与交互深度优化 (2026-04-18 19:31)

## 1. 痛点分析
1. **交互层断层**：以往审计功能 (`backtest_feature_auditor.py`) 作为独立脚本或游离的存在，未能深度整合进每日监盘的主要工具 `instock_MonitorTK.py` 中的表格右键菜单中，操作不够便捷。
2. **批处理缓慢**：单线程下连续对多个具有大背景数据的代码执行特征提取、算分时，需要通过 HDF5 或本地库逐个打榜、读取指标。这在使用图形交互时，表现为严重的界面卡顿，或者是“排队枪毙”。
3. **缺乏承载窗口**：纯终端输出或者控制台打印虽然技术上能够运行，但由于缺少专门为报告进行布局和排版的窗口界面，在日常大量筛选中易用性大减，用户不得不去终端界面提取洞解信息。这显然失去了 Tkinter 实盘盯盘终端“图形化、即得性”的作用。
4. **命名查询冗余**：界面(Treeview)中其实已经渲染了含有各股票的姓名(name)与代码(code)的双维信息，旧接口因为与界面剥离，强行在自身流程中引入 `get_tdx_Exp_day_to_df`、扫描文件匹配名称。

## 2. 解决方案设计与实施
### 步骤 I：重构回测审计引擨的并发与 GUI 展示
在 `backtest_feature_auditor.py` 中引入 `concurrent.futures.ThreadPoolExecutor`：
- 修改原有的批跑函数 `audit_multiple_codes`。当给定了一组任务列表后，并行发放到 8 线程池中独立提取和计算；最后等待合并生成。由于计算核心仅在于算力和磁盘读取而非网络请求，该方案成倍地缩短了多目标验证时间。
- 采用 Tkinter 的核心树形控件 `ttk.Treeview` 以及 `panedWindow` 设计了一个专业的分析报告交互面板：`show_dna_audit_report_window`。其中，上层按综合意图得分 (`Score`) 倒序列表以辅助横向对比个股强弱；下端搭配可滚动的多行文本框显示选中项的具体特征和审计专家洞察。

### 步骤 II：无损接入与防呆保护
在主监测台 `instock_MonitorTK.py` 中：
- 升级 `on_tree_right_click` 处理器，侦听当前的 `self.tree.selection()`。如果右键按下的也是当前高亮的区块之一，不再触发自带重置焦点逻辑；从而无损提取多选列表内所有的 `Code` 与 `Name` 成字典：`code_to_name`。
- 将收集到的列表字典直接以入参形式丢进 `audit_multiple_codes` 中，**消灭了 100% 的姓名数据对盘寻址额外开销**。
- 加设单独守护线程及临时置顶遮罩，当深层次验证运转中（可能由于本地缓存不全消耗数秒读取原始票据），主界面不受阻且提示清楚“审计运转中…”，计算完毕后再回调并唤出 `show_dna_audit_report_window` 给用户结果。
