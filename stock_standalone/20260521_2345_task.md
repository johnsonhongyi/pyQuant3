# 2026-05-21 23:45 Nuitka 编译计时与持久化统计功能

## 任务背景与目标
在 Nuitka 打包编译大型 Python 应用的过程中，编译常常需要花费数分钟到数十分钟不等。为了让用户对每次构建（特别是使用 Clang 优化模式）的编译性能和用时有直观且精准的把控，需要：
1. 在 `nuitka_build_console_onlyClang.bat` 中**添加启动计时**。
2. 在编译完成或失败退出前，**统计并显示总耗时**。
3. 将本次构建的日期、启动时间、结束时间、总用时**持久化到当前路径下的 `time.txt`**。

---

## 实施计划

### 1. 启动计时 (Header Hook)
在 `nuitka_build_console_onlyClang.bat` 顶部的初始化区域（`setlocal enabledelayedexpansion` 之后），通过低侵入的 Python 代码抓取 Unix 时间戳及高辨识度的时间字符串，防止 Windows 批处理原生时间解析的本地化缺陷：
```cmd
for /f "usebackq delims=" %%i in (`python -c "import time; print(time.time())"`) do set "START_TIME=%%i"
for /f "usebackq delims=" %%i in (`python -c "import time; print(time.strftime('%%Y-%%m-%%d %%H:%%M:%%S'))"`) do set "START_TIME_STR=%%i"
```

### 2. 耗时计算与控制台展示 (Footer Hook)
在编译完成的 Verification 后面，但在恢复 `PATH` 与 `exit /b` 之前，通过 Python 计算时间差并格式化输出：
```cmd
for /f "usebackq delims=" %%i in (`python -c "import time; print(time.time())"`) do set "END_TIME=%%i"
for /f "usebackq delims=" %%i in (`python -c "import time; print(time.strftime('%%Y-%%m-%%d %%H:%%M:%%S'))"`) do set "END_TIME_STR=%%i"

for /f "usebackq delims=" %%i in (`python -c "import time; elapsed = %END_TIME% - %START_TIME%; m, s = divmod(elapsed, 60); h, m = divmod(m, 60); print('{:02d}:{:02d}:{:02d} ({:.2f}s)'.format(int(h), int(m), int(s), elapsed))"`) do set "ELAPSED_TIME=%%i"
```

### 3. 持久化至 `time.txt`
在统计完毕后，以追加 (`>>`) 模式将详细的编译概览写入当前工作目录下的 `time.txt` 中。
格式规范如下：
```text
==========================================
Build Date:    2026-05-21 23:45:00
Start Time:    2026-05-21 23:45:00
End Time:      2026-05-21 23:47:15
Elapsed Time:  00:02:15 (135.20s)
==========================================
```
