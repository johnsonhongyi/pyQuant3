## 1. 目标与背景
用户要求在 `kline_monitor.py`（K线趋势实时监控面板）的表格界面中，新增显示 `dff` 与 `dff2` 两列，以方便在实时监控中直接观察行情源的量价偏离信号。

## 2. 实施方案与步骤
根据系统的核心原则（KISS, YAGNI, SOLID），我们采取了微创且高可靠的方案：

1. **表格结构配置扩容 (`__init__`)**：
   - 在 `ttk.Treeview` 的 `columns` 元组中新增 `"dff"` 与 `"dff2"`。
   - 在表头初始化循环中，配置 DFF 与 DFF2 的中文字段名称（分别为 `"DFF"` 和 `"DFF2"`）以及列宽（`35`）。
   - 将窗口默认备用几何尺寸从 `760x460` 优化拓宽至 `860x460`，确保两个新列在不同分辨率或无历史视口记忆时能够完美展示，避免水平挤压。

2. **数据流提取与容错 (`process_table_data`)**：
   - 预先检测底层 `df.columns` 是否包含 `"dff"` 与 `"dff2"` 字段，以极速的 O(1) 布尔标志 `has_dff` / `has_dff2` 缓存，避免在行迭代循环中高频发生 getattr 开销。
   - 行迭代中从源 DataFrame 精准提取对应数据，如果不存在则安全回落为 `0`，杜绝任何 `KeyError` 隐患。

3. **数据渲染与防御型转换 (`update_table`)**：
   - 插入行数据时，对 `dff` 与 `dff2` 进行防御型的 float 类型强制转换，处理所有可能出现的 `None`、`""` 或 `"nan"` 空值，并以 `f"{val:.2f}"` 高精格式化显示。

4. **高级筛选转换 (`apply_filters`)**：
   - 在对筛选语句进行条件评估时，为避免长字符串或隐性数据格式报错，将新列 `"dff"`、`"dff2"` 加入 `pd.to_numeric` 防御转换列表中，确保 pandas 能以高精度数值执行 `.query()` 条件筛选。

## 3. 核心原则的应用体现

- **简单至上 (KISS)**：直接扩展现有的 `ttk.Treeview` 和 `process_table_data` 结构，使用原生的 Tk/ttk 属性和格式化方法，没有引入多余的绘制重构。
- **精益求精 (YAGNI)**：仅实现 `dff` 与 `dff2` 的列呈现、高精格式化、排序以及筛选转换，没有多余的衍生特性，符合 YAGNI 原则。
- **坚实基础 (SOLID)**：
  - **SRP（单一职责）**：数据转换和列设置各司其职，`process_table_data` 专职映射，`update_table` 专职 UI 树装载，各模块职责极其清晰。
- **杜绝重复 (DRY)**：使用统一的遍历映射和行数据解构，充分复用了现有的 Treeview 配置和列排序逻辑。
