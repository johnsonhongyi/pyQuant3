## 1. 任务背景 (Background)
用户在实盘监控交易时，需要极高的人机交互效率。
原先打开“策略选股与确认”界面需要手动点击主面板上的“选股”按钮，且窗口开启后默认停留在第一个选项卡，而核心交易决策数据集中在**“🎯 实时决策”**选项卡。
为了消灭这一操作延迟与割裂，用户提出两个核心诉求：
1. **一键系统级全局热键**：将“选股”界面的开启彻底接入系统原生的 Win32 全局热键架构（RegisterHotKey），即使当前聚焦在 Visualizer、K线主图或外部打字状态，按下 Alt+T 即开。
2. **首屏自动聚焦**：不论是**首次开启**选股窗口，还是在窗口已经处于打开状态下**复用拉起**，窗口开启后均需自动将 Tab 切换至核心的“🎯 实时决策”选项卡，实现一秒读数，极致高效。

---

## 2. 交互与架构设计 (Interactive & Engineering Design)
1. **双通道系统级全局快捷键**：
   - **Win32 级全局热键**：在主程序的 Win32 级全局热键表中，为“策略选股”界面正式注册并绑定系统全局快捷键 `Alt+T`（ASCII键码 `0x54`）。
   - **安全线程调度**：利用 `self._schedule_after(0, self.open_stock_selection_window)` 安全投递至 Tk 主线程，彻底阻断了任何跨线程 GUI 操作死锁隐患。
   - **TK 级兜底本地热键**：依然保留了主控制区 `self.bind_all("<Alt-t>", ...)` 降级通道，确保在特殊平台或无 Win32 GLOBAL 权限时，程序依然能够以 LOCAL 模式完美退避执行。
2. **选项卡流式匹配切换 (Fluid Tab Redirection)**：
   - 在 `StockSelectionWindow` 类中，monkey-patch 植入 `show_decision_tab(self)` 方法。
   - **设计考量**：不写死数字 index，避免未来新增/隐藏其他选项卡时导致界面错位崩溃。
   - **高鲁棒文本扫描**：通过遍历 `self._notebook.tabs()`，利用 `self._notebook.tab(tab_id, "text")` 获取各个 Tab 的标题。只要标题中含有关键字 `"实时决策"`，则瞬时将其 `select(tab_id)` 激活，极具工程前瞻性。
3. **多入口双向联动对齐 (Multi-Entry Alignment)**：
   - **复用链路 (Re-use Linkage)**：在复用已有选股窗口 `_stock_selection_win` 时，在强制 `load_data(force=True)` 后自动调用 `show_decision_tab()`。
   - **新建链路 (New-instance Linkage)**：在全新实例化 `StockSelectionWindow` 后自动调用 `show_decision_tab()`，确保了两大操作入口的行为 100% 严密对称。

## 3. 代码变更详情 (Source Diffs)

### 3.1 选股窗口追加自动切换方法
```python
# stock_selection_window.py
def show_decision_tab(self):
    """切换当前 Notebook 到 🎯 实时决策 选项卡"""
    try:
        if hasattr(self, '_notebook'):
            for tab_id in self._notebook.tabs():
                tab_text = self._notebook.tab(tab_id, "text")
                if "实时决策" in tab_text:
                    self._notebook.select(tab_id)
                    logger.info("📡 选股窗口已自动切换到 🎯 实时决策 选项卡")
                    break
    except Exception as e:
        logger.warning(f"切换到实时决策选项卡失败: {e}")

# monkey-patch 绑定
StockSelectionWindow.show_decision_tab = show_decision_tab
```

## 4. 核心工程原则遵循 (SOLID / KISS)
- **单一职责原则 (SOLID - SRP)**：通过将系统全局物理按键归入 `setup_global_hotkey`，本地窗口内按键归入 `bind_all` 降级，完美保证了双通道架构的高内聚和极强健壮性。
- **简单至上 (KISS)**：采用关键字检索 Tab 而不依赖写死的硬编码 Index，极大地简化了内部引用复杂度，提升了界面的人机交互效率。
