## 解决竞价面板部分个股无分时走势图与惨白单元格的视觉缺陷 (Fixed Bidding Panel Blank Intraday Chart Bug)

### 1. 痛点与根本原因诊断
- **核心异常现象**：中材科技、波长光电、本川智能 等个股的现价（以及涨跌、dff）显示为 `0.00`，且其“分时走势”这一列的单元格中完全为空，没有任何图形或走势线。而沪电股份、深南电路等现价正常的股票却绘制正常。
- **冷启动/竞价无分时数据**：在竞价阶段或今日尚未有撮合成交时，部分个股尚未产生今日的分时线走势（`prices` 数组为空）。
- **Python 字典 API `.get()` 陷阱**：
  - 渲染微型分时图的 `TrendDelegate.paint` 会接受一个 `pdata` 字典，里面显式指定了 `'now_price'` 键，用于在无今日分时数据时 fallback 到该价格绘制一根平整的参考虚线。
  - 原逻辑采用 `now_p = pdata.get('now_price', last_close)` 的方式。
  - 由于数据源在竞价或未交易时返回的最新价为 `0` 或 `0.0`，因此字典中实实在在存在 `'now_price': 0` 键值对。
  - Python 中的 `.get(key, default)` 仅在 `key` 不存在时才返回默认值。当 key 存在且值为 `0` 时，它直接返回 `0`！
- **静默截断**：
  - 于是，`now_p` 被赋予了 `0`。
  - 代码走到 `if now_p > 0` 判定失败，落入 `else` 分支直接 `return` 退出，从而导致绘图逻辑被强行中断，使单元格变成了惨白色且空无一物，给用户带来了界面或数据异常的视觉割裂感。

### 2. 落地解决方案与工程实践
- **引入鲁棒 fallback 机制**：
  - 在 `sector_bidding_panel.py` 与 `tk_gui_modules/qt_table_utils.py` 两处 `TrendDelegate.paint` 的核心渲染逻辑中，升级了对 `'now_price'` 零值/负值的过滤与 fallback 处理：
    ```python
    display_prices = list(prices)
    if not display_prices:
        now_p = pdata.get('now_price', 0)
        if now_p <= 0:
            now_p = last_close
        
        if now_p > 0:
            display_prices = [now_p, now_p]
        else:
            painter.restore()
            return
    ```
  - 这保证了如果获取的最新现价是 `0.0`，系统会智能 fallback 采用昨收价（`last_close`）作为参考现价。
  - 只要个股昨收价大于 0，便可正常画出以昨收价为基底的一条水平参考走势线（昨收平准虚线），完美地占据单元格位置，向用户传达“未交易/竞价开盘前/停牌”的正确状态，彻底消除了视觉大惨白的割裂感，极大提高了界面的工业级专业性。
