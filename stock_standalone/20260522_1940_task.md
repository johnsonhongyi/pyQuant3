## 1. 任务背景 (Background)
用户反馈在实盘自动交易时，存在如下两个极具优化空间的交互与视觉痛点：
1. **高亮自动消失问题**：交易行标记（执行/拦截/错误）会被 15s 一次的周期性刷新 (`_refresh_decision_tab`) 强制抹除重置。用户希望标记**永不自动消失，仅保持高级的“慢闪烁”状态即可**。
2. **悬浮窗死板与不可操作**：此前的 Kernel 执行详情提示仅是一个自动销毁的 Label 弹窗气泡，无法滚动查看且无交互。用户希望**悬浮窗不自动关闭，并且升级为可互动的 Tree 表格视图，点击行能直接与主界面 K 线图及可视化面板进行实时联动**。

---

## 2. 极客化解决方案设计 (Interactive & Aesthetics Design)

### 2.1 高亮行持久化与慢闪烁呼吸灯 (Persistent Highlight & Slow Blink)
- **物理状态沉淀**：在 `StockSelectionWindow` 上维护 `_kernel_marked_exec`、`_kernel_marked_block`、`_kernel_marked_error` 三大状态集合（`set`），将被执行/拦截/出错的代码进行物理存盘记录。
- **刷新短路保护**：重构 `_refresh_decision_tab` 刷新渲染逻辑，当个股代码存在于这些 marked 状态集合中时，采用 Kernel 高亮 tag 强制覆盖普通 tag。**这彻底解决了刷新将高亮擦除的缺陷**。
- **慢闪烁呼吸灯定时器**：实现 `_schedule_kernel_blink` 定时器。每 1500 毫秒交替切换 `kernel_exec` / `kernel_block` / `kernel_error` 的背景色（例如在明色高亮背景与暗色背景间转换，保留前景色高亮），不产生任何多余渲染开销，达成极其惊艳的**慢闪烁呼吸灯视觉效果**。

### 2.2 📡 悬浮 Kernel 交易执行看板 (Interactive Floating Exec Dashboard)
- **免自动销毁**：自动执行完毕后悬浮窗口不自动销毁，允许用户将其停泊在屏幕右侧随时查看，或手动关闭。
- **Treeview 结构化呈现**：在悬浮窗内建 `ttk.Treeview` 表格展示交易明细，包含 `("代码", "名称", "指令", "结果", "详情")` 五列，按绿、黄、红三色标识执行、拦截、异常，质感大幅提升。
- **自愈复用机制**：每次自动执行时自动刷新或重用此看板，清空旧数据并直接载入最新 `records`，绝不会产生多个凌乱弹窗。
- **双向高保真联动**：在 Treeview 行上绑定选择和双击事件，点击即向主界面及可视化面板发送 K 线图联动指令，达成极致一体化联动闭环。

```python
def _schedule_kernel_blink(self):
    """使 kernel_exec/kernel_block/kernel_error 等高亮行慢闪烁"""
    if not hasattr(self, '_signal_tree') or not self._signal_tree.winfo_exists():
        return
    try:
        if not hasattr(self, '_kernel_blink_state'):
            self._kernel_blink_state = True
        self._kernel_blink_state = not self._kernel_blink_state
        if self._kernel_blink_state:
            self._signal_tree.tag_configure("kernel_exec", background="#003a24", foreground="#66ffcc")
            self._signal_tree.tag_configure("kernel_block", background="#3a2600", foreground="#ffcc66")
            self._signal_tree.tag_configure("kernel_error", background="#3a0000", foreground="#ff7777")
        else:
            self._signal_tree.tag_configure("kernel_exec", background="#0a0f1a", foreground="#44ffaa")
            self._signal_tree.tag_configure("kernel_block", background="#0a0f1a", foreground="#e6b800")
            self._signal_tree.tag_configure("kernel_error", background="#0a0f1a", foreground="#ff5555")
        self._kernel_blink_id = self.after(1500, self._schedule_kernel_blink)
    except Exception:
        pass
```

## 4. 核心工程原则遵循 (SOLID / KISS / DRY)
1. **单一职责原则 (SOLID - SRP)**：将交易信号的“物理记录标记”与“动态视图渲染”彻底解耦。`_kernel_mark_signal_rows` 仅负责内存状态修改，而 `_refresh_decision_tab` 专注做一致性绘制。
2. **KISS 原则 (Keep It Simple & Sweet)**：闪烁实现仅通过修改 `tag_configure` 统一属性完成，**无需遍历成千上万个 Item 并分别修改**，耗时为 0.00ms，以极简底层框架爆发了极致流畅的视觉效果。
3. **DRY 原则 (Don't Repeat Yourself)**：整合 `_kernel_show_toast` 窗口自愈机制，杜绝在频繁执行交易时产生多个弹窗。
