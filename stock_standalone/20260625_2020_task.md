# Task: 解决多级排序配置持久化引发的策略选股二次刷新问题

## 目标
解决在进行多级/单列排序后，策略选股窗口（以及其他订阅窗口）高频触发二次刷新、闪烁与选中项抖动的 bug。

## 具体变更
1. **优化全局自选管理器配置文件加载逻辑 (`global_favorites.py`)**：
   - 重构 `GlobalFavoriteManager.load_from_config` 方法。
   - 当检测到配置文件 `window_config.json` 修改时间 (`mtime`) 变动并被重新加载时，只有在“重点板块” (`favorite_sectors`) 或“自选股” (`favorite_stocks`) 发生实际增减变化时，才触发 `notify_subscribers()` 通知订阅者。
   - 对仅因窗口尺寸、分割条比例、表头排序等非自选配置保存导致的配置文件变动进行拦截，不再触发订阅者刷新，从源头上斩断二次重绘的循环。

2. **验证语法与编译安全性**：
   - 运行 python 编译校验工具，确保 `global_favorites.py` 无任何语法、缩进及逻辑错误。
