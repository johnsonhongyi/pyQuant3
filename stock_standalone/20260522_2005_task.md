## 1. 任务背景 (Background)
用户在使用“📡 Kernel 自动交易执行看板”时，经常会根据自身的显示屏和多分屏环境，将此悬浮窗口拖动到特定的位置（如屏幕右下角、副屏等）或调整其尺寸。
原本此窗口每次打开都使用固定的几何位置，无法记忆用户的偏好几何布局。
此外：
1. **级联关闭**：当用户主动或系统自动关闭选股主窗口 (`StockSelectionWindow`) 时，悬浮看板需要同步自动关闭，并在关闭的一瞬间将最新的位置大小进行原子保存。
2. **手动关闭**：当用户单独手动点击悬浮看板右上角系统的“红叉”或者我们自定义的“✕ 关闭”按钮关闭看板时，看板也需要自发自动保存其最新几何参数落盘。
3. **按需置顶控制**：原先看板窗口强制 `-topmost` 置顶在最前方，但在日常分析中这可能会遮挡个股图表。交易员希望**将置顶控制权交给用户，在看板右上角“✕ 关闭”按钮左侧添加一个“📌 置顶”选项卡，勾选时始终在最前，取消勾选时可被其他窗口遮挡，且当前窗口生存期内自动保持勾选状态**。

## 2. 交互与生命周期解决方案设计 (Interactive & Lifecycle Design)
1. **DRY 原则深度复用**：
   - 充分利用了主监控程序 `StockMonitorApp`所继承的 `WindowMixin` 核心方法。
   - `WindowMixin` 提供了成熟的、具有 DPI 自适应和防越界保护的 `load_window_position` 及 `save_window_position` API。
2. **首次自动加载 (Automatic Geometry Loading)**：
   - 在 `win = tk.Toplevel(self)` 创建完毕后，优先通过 `self.master.load_window_position(win, "kernel_toast_window", ...)` 自动载入此前保存的宽高和 `(x, y)`。
   - 若首次启动还没有任何配置文件，自动平滑 fallback 切换至原有的“居右贴合”计算几何体，保证首屏流畅体验。
3. **手/自动置顶按需控制 (`📌 置顶` 复选框)**：
   - **设计考量**：在 `StockSelectionWindow` 上维护一个 `self._kernel_toast_topmost_var = tk.BooleanVar(value=True)`，保证在选股窗口生命周期内能始终记忆用户的置顶勾选状态，默认保持置顶。
   - **高级暗黑系配色 (Dark Rich Aesthetics)**：复选框使用 `bg="#111726"`, `fg="#ffffff"`, `selectcolor="#0c101b"` 进行精心调色，确保其与顶部的深色标题栏和底色浑然一体，绝无亮浅色或刺眼的 Windows 默认边框白底。
   - **一键切换生效**：点击复选框触发 `_toggle_topmost`，实时通过 `win.attributes("-topmost", is_top)` 改变窗口物理属性，实现极高交互效率。
4. **关闭时自动持久化（不使用高频实时记录）**：
   - **设计考量**：废弃在窗口拖动与大小调整过程中的 `<Configure>` 实时监听，避免了在用户拉伸窗口时产生频繁且无意义的文件 IO 写盘开销。
   - **统合手动关闭处理器 (`_on_toast_close`)**：在看板窗口创设时，定义了 `_on_toast_close` 自定义关闭处理方法，统一执行 `save_window_position` 位置大小持久化并最终 `destroy()` 窗口。
   - **物理红叉绑定**：将 `win.protocol("WM_DELETE_WINDOW", _on_toast_close)` 协议绑定，用户点击系统右上角关闭时完美自动写盘。
   - **按钮联动绑定**：将顶部“✕ 关闭”按钮的 `command` 从原先的 `win.destroy` 统一修正并重定向调用 `_on_toast_close`。
5. **级联随同销毁与末期保存 (Cascade Lifecycle Closure & Final Save)**：
   - 深度将看板生命周期嵌入到选股窗口的统一析构入口 `_on_close` 中。
   - 当用户按 ESC 键、点击右上角红叉、或主窗口关闭选股窗口时，`_on_close` 被触发：
     - **末期高保真保存**：首先通过 `self.master.save_window_position(win, "kernel_toast_window")` 再次强制将最新几何信息落盘，万无一失；
     - **物理同步销毁**：紧接着，调用 `win.destroy()` 干净利落地将看板关闭，消除了死悬浮窗残留；
     - **资源释放保护**：同步使用 `self.after_cancel(blink_id)` 注销高亮呼吸灯定时器，彻底根除了 Tk 退出时的后台资源残留与线程/微定时器泄漏。

## 4. 核心工程原则遵循 (SOLID / KISS)
- **开闭原则 (SOLID - OCP)**：通过在 `StockSelectionWindow` 上维护一个 `BooleanVar` 状态变量，在不增加过多全局属性的前提下，精简实现了置顶属性与状态自动保持，易用性极强。
- **简单至上 (KISS)**：采用 TK 原生属性控制，通过 `Checkbutton` 进行直接的命令回调关联，没有任何第三方依赖或复杂的线程通知，极尽简洁健壮。
