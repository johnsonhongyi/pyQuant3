# 修复实时决策状态条显示撑开窗口 Bug 任务清单

> 时间：2026-05-22 19:35

## 任务背景与问题分析
在执行 `_kernel_auto_execute_once`（Kernel自动模拟执行一次）时，系统在计算完交易结果后，会把包含详细操作列表或错误日志的 `detail`（其中包含多个 `\n` 换行符的复杂大文本）和统计汇总信息 `msg` 一同拼接起来发送给 `_kernel_set_status`。
而 `_kernel_set_status` 会调用 `self._kernel_status_lbl.config(...)`，这会导致 `parent`（实时决策 Tab 窗口）顶部高度非常窄的 `risk_bar` 和这一行的 Label 高度暴增，从而强行挤压了下方真正的“实时买点决策队列”表格（`self._signal_tree`）、持仓及今日流水等主表格控件，使原本设计紧凑美观的窗口排布结构瞬间变形。

## 任务解决方案
1. **单行化状态条信息 (Status Bar Simplification)**：
   将 `_kernel_auto_execute_once` 中 `self._kernel_set_status(...)` 所设置的文本仅保留为极其简洁的单行摘要信息 `msg`（如 `"执行=1 拦截=0 错误=0"` ），完全剔除带有 `\n` 的大长文本 `detail` 传递，确保状态条高度 100% 保持稳定。
2. **气泡提示留存 (Toast Details Preservation)**：
   保留原有在 `_kernel_show_toast` 中传递 `detail` 的逻辑。这可以让用户依然看到最顶层透明悬浮框（2.6秒自动销毁）中显示的详细买卖操作详情与错误码，确保系统操作的透明性和强交互可解释性，同时又不破坏任何主界面的美观排布。

## 修改内容

```diff
-    self._kernel_set_status(msg + (f" | {detail}" if detail else ""), kind)
+    # [FIX] 状态栏仅显示单行简洁摘要，避免 detail 换行撑开高度改变 UI 窗口排布
+    self._kernel_set_status(msg, kind)
```
