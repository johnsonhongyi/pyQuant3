# Task: DNA专项审计功能深度集成优化

## 目标
整合 `backtest_feature_auditor.py` 的“DNA 专项审计”能力到主监测界面 (`instock_MonitorTK.py`) 的右键菜单中，支持批量选中多只股票进行高性能分析，并提供专属的分析报告窗口，不再遗失在控制台后台打印中。

## 具体变更
1. **Tkinter 右键菜单集成与多选批处理**
   - 优化 `instock_MonitorTK.py` 中的 `on_tree_right_click` 方法，使其能正确捕获当前选中的多条数据，而不直接重置单选。
   - 提取已经获取过的选定个股的 `code` 和 `name` 的映射字典，从而实现在调用深层审计器时无需再次进行耗时的名称数据库匹配及查询。

2. **多线程并发提速 (批量极限挖掘)**
   - 彻底优化原有的单次迭代运行逻辑，将其引入 `concurrent.futures.ThreadPoolExecutor(max_workers=8)`。
   - 从原先 1-2秒 / 股票降低至总共处理耗时不随批量大小近线性递减的问题，实现无阻塞极限性能。

3. **专属视图架构**
   - 打造具有列表排名（包含代码、名称、DNA得分、波段涨幅和结果判定）的全新 `Toplevel` 分析界面：`show_dna_audit_report_window`。
   - 具有双区域自适应布局 (Top Table + Bottom ScrolledText Detail)，双击/单击任何列表项可直接洞察核心报告内容。
   - 兼容旧的底层代码并在执行过程中添加交互式“执行中...”防呆浮窗。
