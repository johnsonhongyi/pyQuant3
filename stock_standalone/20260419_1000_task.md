# 20260419_1000_task.md - Fix MonitorTK AttributeError

## 目标
修复 `instock_MonitorTK.py` 中 `send_df` 线程在启动时可能遭遇的 `AttributeError: '_df_sync_running'` 异常。

## 问题分析
1. `send_df` 线程在循环中直接访问 `self._df_sync_running`。
2. 该变量是在 `_start_feedback_listener` 中初始化的，而 `_start_feedback_listener` 是通过 `after(3000, ...)` 延迟启动的。
3. 如果在启动后的前 3 秒内，通过 `_start_visualizer_process`（手动点击按钮或联动触发）提前启动了同步线程，此时 `_df_sync_running` 尚未定义，导致崩溃。
4. 此外，`_start_visualizer_process` 自身在启动线程前也遗漏了将该标志设为 `True` 的逻辑。

## 修复步骤
- [x] 在 `StockMonitorApp.__init__` 中显式初始化 `_df_sync_running = False`。
- [x] 在 `StockMonitorApp.__init__` 中同步初始化 `_df_first_send_done` 和 `_force_full_sync_pending` 标志位。
- [x] 在 `_start_visualizer_process` 启动 `send_df` 线程前，确保设置 `self._df_sync_running = True`。

## 验证计划
- 启动应用，在 3 秒内立即点击 📊 按钮启动可视化。
- 观察控制台是否仍有 `AttributeError` 报错。
