# 数据库修复 - 快速参考

## 🚀 一键修复

```bash
# 修复默认数据库
python instock_MonitorTK.py -repair-db
```

## 📖 常用命令

| 命令 | 说明 |
|------|------|
| `python instock_MonitorTK.py -repair-db` | 修复默认数据库 (signal_strategy.db) |
| `python instock_MonitorTK.py -repair-db mydb.db` | 修复指定数据库 |
| `python instock_MonitorTK.py --help` | 查看所有命令行参数 |
| `instock_MonitorTK.exe -repair-db` | EXE 版本修复数据库 |

## ✅ 使用步骤

1. **关闭主程序** - 确保数据库未被占用
2. **运行修复命令** - 执行上述命令之一
3. **等待完成** - 查看输出信息
4. **检查结果** - 确认修复成功

## 📁 文件说明

- `signal_strategy.db` - 主数据库文件
- `signal_strategy.db.backup_*` - 自动创建的备份文件
- `signal_strategy.db.new` - 重建的新数据库(如果需要)
- `instock_tk.log` - 日志文件

## ⚡ 快捷方式

### Windows 批处理文件

创建 `修复数据库.bat`:
```batch
@echo off
cd /d "%~dp0"
python instock_MonitorTK.py -repair-db
pause
```

双击运行即可!

### PowerShell 脚本

创建 `修复数据库.ps1`:
```powershell
Set-Location $PSScriptRoot
python instock_MonitorTK.py -repair-db
Read-Host "按回车键退出"
```

## 🔧 故障速查

| 问题 | 解决方案 |
|------|----------|
| 文件不存在 | 检查路径是否正确 |
| 数据库被锁定 | 关闭主程序后重试 |
| 磁盘空间不足 | 清理磁盘空间 |
| 修复失败 | 使用备份文件恢复 |
