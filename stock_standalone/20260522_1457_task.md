# 任务清单: 修复放量详情个股弹窗表格排序功能失效问题 (Fixed VolumeDetailsDialog Sorting Bug)

## 任务背景 (Background)
用户反馈在 `signal_dashboard_panel.py` 的第 353-368 行附近的放量详情个股弹窗 (`VolumeDetailsDialog`) 中，表格排序功能失效。

经过审查代码发现，系统中在向 `VolumeDetailsDialog` 写入新数据时，为避免数据错位，在 `update_data` 入口暂时关闭了排序。但是在数据填充完毕的收尾步骤以及初始化时，不慎将 `self.table.setSortingEnabled(True)` 误写成了 `self.table.setSortingEnabled(False)`。这直接导致了该表格在日常实盘运行中，其排序功能被彻底静默关闭。

## 解决方案与工程落地 (Implementation)

1. **修正初始化设置**：
   - 将 `VolumeDetailsDialog.__init__` 中的 `self.table.setSortingEnabled(False)` 修复为 `self.table.setSortingEnabled(True)`。这与 `MarketAlertDetailDialog` 等成熟组件保持完全一致。

2. **数据装载 try-finally 闭环加固**：
   - 在 `VolumeDetailsDialog.update_data` 中，引入了高可靠的 `try-finally` 结构：
     - 在 `try` 块最前端继续通过 `self.table.setSortingEnabled(False)` 暂时关闭排序，以获得最高的数据插入性能，防止写入过程中触发重新排序导致的界面错乱。
     - 在 `finally` 块中强制将其恢复为 `self.table.setSortingEnabled(True)`，并确保 `self._is_updating = False` 和 `setSortIndicatorShown(True)` 被绝对执行，即使在数据装载中发生偶发性异常也不会导致状态机锁死。

3. **双向无损支持**：
   - 与 `NumericTableWidgetItem` 的排序完美联动。涨幅 `%` 和量比这两个数值列能按实际数值的高低大小进行完美的升降序排列，而不是按普通的 ASCII 字符串排序。
