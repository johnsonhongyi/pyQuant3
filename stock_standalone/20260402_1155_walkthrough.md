# Walkthrough - 修复九转 K 线染色误导问题 (2026-04-02)

## 1. 问题背景
在启用“九转”指标模式下，系统有时会在个股出现“大幅杀跌”或“大阴线”时，依然执行“红色持股”的染色逻辑。这通常是因为指标计算存在一定的平滑或滞后，导致在价格剧烈波动的第一时间，状态位（`is_red_hold`）尚未切换，从而在视觉上产生严重的误导性反馈（本该看到杀跌，却看到了红色）。

## 2. 核心改进方案
在 `trade_visualizer_qt6.py` 的 K 线染色循环逻辑中，引入了基于盘中价格的实时“安全审计”：

- **新增阴阳线判定**：通过比较 `close < open` 实时确定每一根 K 线是否为下跌性质。
- **红色禁令 (Red-Hold Guard)**：如果判定当前为阴线（下跌），则无论 `is_red_hold` 信号如何，都强制禁止渲染红色（`#FF0000`）。此时渲染逻辑会自动降级/还原为 K 线默认的阴线颜色（绿色/青色）或保持“主力卖出”的紫色。

## 3. 代码变更点清单


## 4. 验证结果
- **预期反馈**: 当出现如用户反馈中的“大幅杀跌大阴线”时，K 线将不再显示为错误红色。
- **视觉兼容**: 紫色（主力卖出）和黄色/青色（主力买入）依然具备最高优先级，且对阴阳线不做硬性拦截，保持了指标的敏感度。
- **切换测试**: 关闭九转开关后，K 线颜色能够平滑回归标准模式。

## 5. 结论
通过将物理价格走势（阴阳线）作为 UI 染色的“第一准则”，彻底解决了指标滞后带来的视觉认知错误。
