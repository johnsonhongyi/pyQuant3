# 任务清单 (2026-05-22 16:25) - 修复放量详情及预警明细弹窗视图不同步拉伸/缩放 Bug & 扩展展示 DFF 与 DFF2 列 & 扩增放量个股容量至 Top 200

## 1. 问题分析与诊断 (Problem Diagnosis)
- **现象 1 (Symptom 1 - UI Scaling)**：
  当用户拖动放大或缩小个股异动放量详情弹窗 (`VolumeDetailsDialog`) 或个股预警详情弹窗 (`MarketAlertDetailDialog`) 时，内部的列停留在左侧固定大小，右侧在放大时显示大片黑色/深蓝色的空白背景。用户反馈“是两个分开显示，需要跟其他视图一样，里面的列同步放大缩小保持一致性，而不是两层显示”。
- **现象 2 (Symptom 2 - Missing DFF Columns)**：
  用户希望像 `MarketAlertDetailDialog` 一样，在个股异动放量详情弹窗 (`VolumeDetailsDialog`) 中增加 `DFF` 与 `DFF2` 两列显示，以对齐多维度量化指标监控。
- **现象 3 (Symptom 3 - Limited Sorting Pool)**：
  原今日异动放量个股弹窗写死且只推送了前 30 个标的 (`Top 30`)，这导致用户在界面点击表头（如按涨幅、量比、DFF）排序时，永远都只是在初始提取的这 30 个标的里打转，局限性极大。

- **根源 (Root Cause)**：
  1. **底层句柄重建导致状态脱节 (HWND Destruction / Re-creation Trap)**：
     物理改变 WindowFlags 会销毁当前的窗口句柄（HWND）并重新创建一个新句柄，导致之前的位置状态重置。
  2. **列宽未设为 Stretch 模式**：
     原先的表头列宽是交互式 (`Interactive`) 或固定值，拉宽窗口时没有平分，导致视觉割裂。
  3. **数据源缺失与 UI 未声明**：
     `VolumeDetailsDialog` 的数据来源 `vol_details`（由 `instock_MonitorTK.py` 生成并派发）原先只提取了 `"code"`, `"name"`, `"change"`, `"ratio"`，未提取 `dff` 与 `dff2` 列数据，同时 `VolumeDetailsDialog` 的 `QTableWidget` 只声明了 4 列，导致两端指标脱水。
  4. **固化的 `head(30)` 截断**：
     主行情数据源中使用了 `df[vol_mask].head(30)` 对放量个股列表进行了硬性截断，掐断了后方海量潜力个股的流入。

## 2. 解决方案与实施步骤 (Implementation Steps)

1. **扩充标的池容量至 Top 200 (Expanded Capacity to Top 200)**：
   将 `instock_MonitorTK.py` 中的 `vol_mask` 截取限制提升到 200 行：
   `sub_df = df[vol_mask].head(200)`
   并同步更新 `VolumeDetailsDialog` 的窗口标题为 `"🔥 今日异动放量个股 (Top 200)"`。
   - **效果**：每次行情推送将足额携带多达 200 个真实的放量个股。用户点击表头进行“涨幅”、“量比”、“DFF”等指标升降序排序时，数据会在 200 个优质个股中全量重排，大幅度提升了盘中大池子挖掘和分析的实战价值！

2. **表格结构配置扩容与首屏布局自适应 (Table Structure Expansion)**：
   将 `VolumeDetailsDialog` 的 `QTableWidget` 扩充为 6 列，表头重新定义为：
   `["代码", "名称", "涨幅%", "量比", "DFF", "DFF2"]`。
   在全新的等宽自适应拉伸模式 `QHeaderView.ResizeMode.Stretch` 下，这 6 列在窗口改变大小时能够百分之百保持一体化同步放大缩小，极度流畅美观。

3. **安全数据路由与 NaN 兜底过滤 (Robust Data Generation & Fallback Pipeline)**：
   在 `instock_MonitorTK.py` 的数据推送循环中，精准抓取 `dff` 与 `dff2` 列数据。为了绝对防止 `NaN` 或 `None` 值写入字典后引发的 JSON 传输序列化崩溃，引入了高可靠的类型检测与兜底自愈过滤：
   ```python
   dff_val = row.get('dff', 0.0)
   dff2_val = row.get('dff2', 0.0)
   dff_val = 0.0 if (dff_val is None or pd.isna(dff_val)) else float(dff_val)
   dff2_val = 0.0 if (dff2_val is None or pd.isna(dff2_val)) else float(dff2_val)
   ```

4. **数值高精格式化与精准排序对齐 (Formatted Rendering & Smart Sorting)**：
   使用我们之前封装的高效 `NumericTableWidgetItem` 承载 `DFF` 与 `DFF2`：
   - 格式化渲染为两位小数（保留正负号，如 `+1.23` 或 `-0.45`），展示更贴切。
   - 对大于 0 的数值涂覆亮红色警示 (`#ff4444`)，小于 0 的数值涂覆明绿色 (`#44ff44`)。
   - 完美适配了表头点击时的数值型升降序精确排序，杜绝字符串型排序错乱。

5. **物理强力加固 `resizeEvent` (Hardened resizeEvent)**：
   在两个弹窗内部均强力重写了 `resizeEvent`，物理强制重置布局管理器几何大小以同步 `self.rect()`，消除“两层显示”缺陷。
