# 清理非交易时段信号 - 命令行使用说明

## 快速开始

### 方式一: 通过主程序命令行参数(推荐,打包后可用)

```bash
# 1. 清理信号数据库(默认)
python instock_MonitorTK.py -cleanup-signals

# 或打包后
instock_MonitorTK.exe -cleanup-signals

# 2. 清理所有数据库
python instock_MonitorTK.py -cleanup-signals all

# 3. 清理指定数据库
python instock_MonitorTK.py -cleanup-signals trading_signals.db
python instock_MonitorTK.py -cleanup-signals signal_strategy.db
```

### 方式二: 直接运行清理脚本(开发环境)

```bash
# 试运行查看统计(不删除)
python cleanup_non_trading_signals.py --dry-run

# 清理所有数据库
python cleanup_non_trading_signals.py --all

# 清理指定数据库
python cleanup_non_trading_signals.py --db signal_strategy.db
```

## 详细说明

### 命令行参数说明

**`-cleanup-signals [数据库]`**

- **不带参数**: 默认清理 `signal_strategy.db`
- **`all`**: 清理所有数据库(`trading_signals.db` 和 `signal_strategy.db`)
- **指定数据库**: 清理指定的数据库文件

### 清理范围

**trading_signals.db**:
- 表: `live_signal_history`
- 清理非交易时段的实时信号记录

**signal_strategy.db**:
- 表: `signal_message`
- 清理非交易时段的策略信号记录

### 交易时段定义

**必须同时满足**:
1. 交易日(通过 `cct.get_trade_date_status()` 判断)
2. 交易时段(09:30-11:30 或 13:00-15:00)

**非交易时段**(将被清理):
- 非交易日(周末、节假日)
- 集合竞价时段(09:15-09:30)
- 午间休市(11:30-13:00)
- 收盘后(15:00之后)
- 开盘前(09:30之前)

### 安全措施

1. **自动备份**: 清理前会自动备份数据库,文件名格式: `数据库名.backup_YYYYMMDD_HHMMSS`
2. **确认提示**: 如果备份失败,会提示是否继续
3. **详细日志**: 显示清理进度和结果统计

## 验证清理结果

### 方法1: 查询数据库

```sql
-- 检查 signal_message 表的时间分布
SELECT 
    substr(timestamp, 12, 5) as time,
    COUNT(*) as count
FROM signal_message
GROUP BY substr(timestamp, 12, 5)
ORDER BY time;

-- 应该只看到 09:30-11:30 和 13:00-15:00 的记录
```

### 方法2: 查看备份文件

清理后会生成备份文件,可以对比备份文件和当前数据库的大小:

```bash
# 查看文件大小
dir *.db*

# 示例输出:
# signal_strategy.db                  2,345,678 字节
# signal_strategy.db.backup_20260211  3,456,789 字节  (更大,包含非交易时段数据)
```

## 定期维护建议

### 手动清理

建议每周或每月运行一次:

```bash
python instock_MonitorTK.py -cleanup-signals all
```

### 自动清理(可选)

可以通过 Windows 任务计划程序设置定期清理:

1. 打开"任务计划程序"
2. 创建基本任务
3. 触发器: 每周一次
4. 操作: 启动程序
   - 程序: `python.exe` 或 `instock_MonitorTK.exe`
   - 参数: `instock_MonitorTK.py -cleanup-signals all` 或 `-cleanup-signals all`
   - 起始于: 程序所在目录

## 注意事项

1. **备份重要**: 虽然程序会自动备份,但建议手动保留重要数据的额外备份
2. **关闭程序**: 清理前确保主程序已关闭,避免数据库锁定
3. **磁盘空间**: 备份文件会占用磁盘空间,定期清理旧备份
4. **验证结果**: 清理后建议查询数据库验证结果

## 故障排查

### 问题1: 数据库文件不存在

**错误**: `⚠️ 数据库文件不存在,跳过: xxx.db`

**解决**: 
- 确认数据库文件路径正确
- 确认在正确的目录下运行命令

### 问题2: 数据库被锁定

**错误**: `database is locked`

**解决**:
- 关闭所有访问数据库的程序
- 等待几秒后重试

### 问题3: 备份失败

**错误**: `❌ 备份失败: xxx`

**解决**:
- 检查磁盘空间是否充足
- 检查文件权限
- 手动备份后使用 `--no-backup` 参数(仅脚本方式)
